# WhisperJAV-Auto 使用说明

WhisperJAV-Auto 是一个 Windows PowerShell 自动化封装，用于：

```text
视频 -> WhisperJAV Ensemble 双 Pass 日语转录 -> DeepSeek 中文翻译 -> 视频同名 .srt
```

本工具只是自动化封装，底层转录依赖 WhisperJAV：

```text
WhisperJAV 官方仓库：https://github.com/meizhong986/WhisperJAV
```

## 目录结构

用户日常只需要关注根目录这些文件：

- `开启中文字幕自动处理.bat`：开启自动处理并打开任务进度窗口。
- `关闭中文字幕自动处理.bat`：关闭自动处理并停止相关进程。
- `查看中文字幕处理状态.bat`：查看当前任务、最近日志和失败清单数量。
- `config.example.json`：配置模板，复制一份后改名为 `config.json`。
- `config.json`：你自己的实际配置文件，按本机环境修改；默认不会提交到 Git。
- `使用说明.md`：本文档。

`internal` 是内部脚本目录，正常不要修改。

## 使用 BAT 前必须完成的配置

不要一开始就双击 BAT。第一次使用请先按下面顺序配置和验证。

### 1. 准备 WhisperJAV

先从 WhisperJAV 官方仓库安装 WhisperJAV：

```text
https://github.com/meizhong986/WhisperJAV
```

先确认 WhisperJAV 已经安装好，并且目录里能找到这些文件：

```text
D:\whisperjav\Scripts\whisperjav.exe
D:\whisperjav\Scripts\whisperjav-translate.exe
D:\whisperjav\ffmpeg\ffprobe.exe
```

如果你的 WhisperJAV 安装在别的位置也可以，但后面 `config.json` 里的 `paths.whisperJavInstallDir` 必须改成实际目录。

建议先在 PowerShell 里测试 WhisperJAV 能否运行：

```powershell
D:\whisperjav\Scripts\whisperjav.exe --help
D:\whisperjav\Scripts\whisperjav-translate.exe --help
```

如果要确认 GPU 版 PyTorch 是否正常：

```powershell
D:\whisperjav\python.exe -c "import torch; print(torch.__version__); print(torch.cuda.is_available()); print(torch.cuda.get_device_name(0))"
```

正常应看到：

```text
True
你的 NVIDIA 显卡名称
```

如果 `torch.cuda.is_available()` 是 `False`，先修好 WhisperJAV / CUDA / PyTorch，再使用本工具。

### 2. 创建本机配置文件

复制配置模板，生成自己的配置文件：

```powershell
Set-Location D:\WhisperJAV-Auto
Copy-Item .\config.example.json .\config.json
```

然后编辑 `config.json`，按自己的视频目录、工作目录、WhisperJAV 安装目录和 DeepSeek 设置修改。

`config.json` 默认已被 `.gitignore` 排除，不会上传到公开仓库。

### 3. 配置视频目录和工作目录

主要修改 `paths.videoRoots`、`paths.workRoot`、`paths.whisperJavInstallDir`。

```json
"paths": {
  "workRoot": "E:\\WhisperJAV-Auto\\work",
  "manualOutputDir": "D:\\WhisperJAV-Auto\\manual-output",
  "videoRoots": [
    "X:\\videos"
  ],
  "nasDrivePrefix": "X:\\",
  "nasUncPrefix": "\\\\SERVER\\Share\\",
  "whisperJavInstallDir": "D:\\whisperjav"
}
```

字段说明：

- `workRoot`：本机临时工作目录，建议放 SSD 或本机硬盘，不要放 NAS。
- `manualOutputDir` 用于目标目录不可写时保留字幕。
- `videoRoots`：要扫描的视频目录，可以填多个。
- `nasDrivePrefix`：映射盘前缀，例如 `X:\`。
- `nasUncPrefix`：对应 UNC 路径，例如 `\\SERVER\Share\`。
- `whisperJavInstallDir`：WhisperJAV 安装目录。

示例：如果视频在这些目录：

```text
X:\downloads\AV
X:\Other
```

就写成：

```json
"videoRoots": [
  "X:\\downloads\\AV",
  "X:\\Other"
]
```

### 4. 配置 DeepSeek

推荐使用环境变量保存 API Key：

```powershell
setx DEEPSEEK_API_KEY "sk-xxxxxxxx"
```

也可以直接写入 `config.json`：

```json
"deepSeek": {
  "apiKey": "sk-xxxxxxxx"
}
```

公开仓库不要提交带真实 API Key 的 `config.json`。

### 5. 确认转录参数

默认配置是 Ensemble 双 Pass 质量优先模式：

```text
Pass1: fidelity / balanced / auditok / ten / large-v2
Pass2: balanced / aggressive / silero / silero-v6.2 / large-v2
Merge: pass1_primary
```

这些参数在 `config.json` 的 `transcription` 节点里。确认 WhisperJAV 版本支持这些参数后再运行。

如果你不确定版本是否支持，可以测试：

```powershell
D:\whisperjav\Scripts\whisperjav.exe --help
```

### 6. 手动做一次配置检查

在 PowerShell 里执行：

```powershell
Set-Location D:\WhisperJAV-Auto
powershell.exe -NoProfile -ExecutionPolicy Bypass -File .\internal\Status-WhisperJAV-Auto.ps1
```

确认输出里能看到：

```text
DeepSeek 模型
目标语言
翻译风格
最近日志
```

### 7. 再双击 BAT

配置确认后再使用 BAT：

- 双击 `开启中文字幕自动处理.bat`
- 观察新开的任务进度窗口
- 需要停止时双击 `关闭中文字幕自动处理.bat`

## 日常使用

开启：

```text
双击 开启中文字幕自动处理.bat
```

关闭：

```text
双击 关闭中文字幕自动处理.bat
```

查看状态：

```text
双击 查看中文字幕处理状态.bat
```

## 跳过规则

脚本会跳过：

- 已有同名外挂字幕的视频。
- MKV 内嵌字幕的视频。
- 文件名包含中文字幕标记的视频，例如 `ABF-159-C.mp4`、`CAWD-590-C_GG5.mp4`、`NSFS-400ch.mp4`。
- 失败清单里的损坏视频。

## 日志与输出

日志目录：

```text
logs
process-logs
```

正常输出：

```text
影片.mp4
影片.srt
```

如果目标视频目录不可写，中文字幕会保留到：

```text
manual-output
```
